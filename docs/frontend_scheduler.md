# 前端定时任务功能整合

## 🎉 功能概述

已将定时自动爬取功能完整整合到前端 UI 中，现在可以通过网页界面直接控制定时任务。

## ✨ 新增功能

### 1. 定时任务控制面板

在"热榜评论分析"页面中，新增了定时任务控制区域，包括：

- **爬取间隔设置**：支持 1/2/4/6/12/24 小时
- **启动/停止按钮**：一键控制定时任务
- **实时状态显示**：显示当前运行状态和统计信息

### 2. 实时状态监控

定时任务状态每 5 秒自动刷新，显示：

- ✅ 运行状态（运行中/未运行）
- ⏰ 下次运行时间
- 📊 总运行次数
- 💬 总评论数
- 🔢 配置参数（间隔、视频数、评论数）

### 3. 可视化界面

- 🟢 **运行中**：绿色背景，旋转图标
- ⚫ **未运行**：灰色背景，静态图标
- 📈 **统计数据**：网格布局，清晰展示

## 🚀 使用步骤

### 步骤 1：打开热榜评论分析页面

1. 启动前端应用
2. 点击左侧菜单"热榜评论"
3. 进入热榜评论分析页面

### 步骤 2：配置爬取参数

在"爬取配置"区域设置：

- **爬取视频数量**：1-30 个（默认 10 个）
- **每个视频评论数**：10-1000 条（默认 100 条）

### 步骤 3：启动定时任务

1. 在"定时自动爬取"区域选择爬取间隔
   - 1 小时：适合高频监控
   - 2 小时：推荐，平衡数据和资源
   - 4-24 小时：适合长期观察

2. 点击"启动定时任务"按钮
   - 按钮变为紫色
   - 状态面板显示"定时任务运行中"
   - 显示下次运行时间

### 步骤 4：监控运行状态

状态面板实时显示：

```
🟢 定时任务运行中
├─ 间隔时间：2 小时
├─ 视频数量：10
├─ 评论数/视频：100
├─ 总运行次数：5
├─ 上次运行：2026-03-26 23:51:20
├─ 下次运行：2026-03-27 01:51:20
└─ 总评论数：4925
```

### 步骤 5：停止定时任务

点击"停止定时任务"按钮：
- 按钮变为红色
- 状态面板显示"定时任务未运行"
- 保留统计数据

## 🔧 技术实现

### 后端 API

新增 3 个 API 接口：

1. **启动定时任务**
   ```
   POST /api/hot-comment/scheduler/start
   Body: {
     "interval_hours": 2,
     "video_count": 10,
     "comments_per_video": 100
   }
   ```

2. **停止定时任务**
   ```
   POST /api/hot-comment/scheduler/stop
   ```

3. **获取状态**
   ```
   GET /api/hot-comment/scheduler/status
   ```

### 前端组件

**状态管理：**
- `schedulerRunning`: 运行状态
- `schedulerStatus`: 详细状态数据
- `intervalHours`: 间隔时间设置

**自动刷新：**
- 每 5 秒自动获取最新状态
- 实时显示运行统计

**UI 组件：**
- 下拉选择框：间隔时间
- 启动/停止按钮：带图标和动画
- 状态面板：动态颜色和数据

## 🎯 最佳实践

### 1. 首次使用

建议先手动爬取一次：
1. 点击"开始爬取"
2. 确认功能正常
3. 再启动定时任务

### 2. 选择合适的时间间隔

- **1-2 小时**：实时监控，数据量大
- **4-6 小时**：日常收集，适中
- **12-24 小时**：长期观察，数据量小

### 3. 定期检查

- 查看状态面板确认运行正常
- 检查输出目录确认文件生成
- 查看日志确认无错误

### 4. 资源管理

- 晚上可停止或延长间隔
- 定期清理旧 CSV 文件
- 注意磁盘空间

## 🔍 故障排除

### 问题 1：启动失败

**现象：** 点击启动按钮无反应

**解决：**
1. 检查 Cookie 是否有效
2. 查看浏览器控制台错误
3. 检查后端日志

### 问题 2：状态不更新

**现象：** 状态面板一直显示"未运行"

**解决：**
1. 刷新页面
2. 检查网络连接
3. 查看 API 是否正常

### 问题 3：定时任务停止

**现象：** 运行中突然停止

**解决：**
1. 查看后端日志
2. 检查是否有异常
3. 重新启动定时任务

## 📝 注意事项

1. **Cookie 有效期**：Cookie 过期会导致爬取失败
2. **网络稳定性**：需要稳定的网络连接
3. **系统资源**：运行时占用少量 CPU 和内存
4. **磁盘空间**：定期清理旧文件
5. **浏览器刷新**：刷新页面后状态会自动恢复
